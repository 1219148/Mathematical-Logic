# RL Agent 阶段报告

## 目标

本阶段目标是把 task5 纳入 RL agent 的训练与评测范围，同时避免两类问题：

1. 默认运行时依赖 `submissions/student_agent.py` 兜底。
2. 为 task5 写一条固定路线，造成过拟合或复制教师轨迹的嫌疑。

## 已完成内容

- 默认入口 `rl_agent.q_learning_agent` 指向独立的 options RL policy，不在线导入 `student_agent`。
- 新增/保留 `train_options_q.py`，默认训练集合为 task1-task5。
- `options_policy.py` 使用通用 option 空间：开箱、战斗、按钮/开关、出口探索。
- 加入了跨房间打断、已阻塞格记忆、怪物危险区避让等通用机制，避免把 task5 房间名写入策略。

## 当前评测结论

task1-task4 在此前验证中已经可以通过。task5 已纳入训练与评测，但当前还没有稳定通关。

当前 task5 的稳定进展包括：

- 起点房间开箱拿金币。
- 按下按钮打开南侧条件出口。
- 进入南房间并打开钥匙箱。
- 返回起点后打开东侧锁门。
- 进入东房间并打开治疗箱。

未解决瓶颈：

- 从东房间返回与继续探索西房间时，出口定位和移动持续动作仍会消耗大量步数。
- task5 有健康随时间衰减的压力，当前策略到达西房间前血量过低。
- 高层 option-Q 训练还需要进一步 shaping，使其学习“少战斗、少绕路、优先完成剩余箱子”的策略。

## 不过拟合说明

当前没有加入 `task_5` 专属房间名、坐标路线或教师轨迹 replay。所有改动都基于局部状态：可见箱子、怪物、按钮、出口、房间访问、阻塞格和库存。task5 被作为训练任务之一纳入 `train_options_q.py`，而不是单独写策略分支。

## 下一步建议

1. 改进 `train_options_q.py` 的奖励 shaping：新增箱子、开门、换房、治疗给予稀疏正奖励，对无进展循环和低血量耗时给惩罚。
2. 在 option 状态中加入健康分桶和全局剩余箱子记忆，帮助 task5 学会更短路线。
3. 修正出口 tile 在玩家站上去后从 `state.exits` 消失导致的边界振荡问题。
4. 重新训练 task1-task5，并分别评测加载 Q 表与禁用 Q 表两种模式，确认提升来自 RL option 选择而不是手工 task5 路线。
